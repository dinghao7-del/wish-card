## 晚间追加：头像系统彻底修复（23:45）

### 问题根因
- 所有外部头像 CDN（Unsplash、DiceBear）在国内无法访问，导致 `<img>` 加载失败
- `AvatarSelector.tsx` 返回 `text:字符,颜色,颜色` 格式字符串，不是合法 URL
- `FamilyContext.tsx` 默认头像使用 `getDiceBearUrl()` 生成外部 URL

### 修复方案：100% 本地 data URL 头像
所有头像改为 `data:image/svg+xml,...` 格式，完全不需要网络：

1. **`templates.ts`** — 文字头像数组的 `url` 字段在定义时直接生成 data URL
2. **`AvatarSelector.tsx`** — 用 `<div>` 渲染文字头像（备用方案），确认按钮返回 data URL
3. **`FamilyContext.tsx`** — 默认头像改为 `nameToDataUrl(name)` 生成，加载时强制执行

### 关键代码
```typescript
// 生成文字头像的 Data URL（本地 SVG，无需网络）
const nameToDataUrl = (name: string): string => {
  const char = name.charAt(0);
  const colors = ['#FF6B6B','#FFA726','#66BB6A','#42A5F5','#AB47BC','#EC407A','#26C6DA','#8D6E63'];
  let hash = 0;
  for (let i = 0; i < name.length; i++) {
    hash = name.charCodeAt(i) + ((hash << 5) - hash);
  }
  const bgColor = colors[Math.abs(hash) % colors.length];
  const textColor = ['#FFEE58','#FFF176','#66BB6A'].includes(bgColor) ? '#333' : '#FFF';
  const svg = `<svg xmlns="http://www.w3.org/2000/svg" width="100" height="100" viewBox="0 0 100 100"><rect width="100" height="100" rx="50" fill="${bgColor}"/><text x="50" y="68" text-anchor="middle" fill="${textColor}" font-size="55" font-weight="bold">${char}</text></svg>`;
  return `data:image/svg+xml,${encodeURIComponent(svg)}`;
};
```
