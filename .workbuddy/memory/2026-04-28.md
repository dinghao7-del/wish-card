## Bug 修复：欢迎页卡死 + 潜在无限循环

### P0 Bug 修复：欢迎页无法关闭 ✅

**根因**：React `setState` 异步更新后立即 `navigate('/')`，但 Layout 在导航瞬间 `currentUser` 仍为 null，触发重定向回 /welcome。

**修复方案**：在 `Welcome.tsx` 添加 `useEffect` 监听 `currentUser` 变化，统一在状态更新后导航。

```tsx
useEffect(() => {
  if (currentUser) {
    navigate('/', { replace: true });
  }
}, [currentUser, navigate]);
```

**同时修复**：
- `handleSkip` - 已修复（跳过了直接 navigate）
- `handleRegister` - 已修复（移除了 setTimeout 中的 navigate）
- `handleLogin` - 已修复（移除了 setTimeout 中的 navigate）

### Home.tsx useEffect 潜在无限循环修复 ✅

**根因**：`prevStars` 使用 `useState` 且放在 `useEffect` 依赖数组中，状态更新触发 effect → 状态更新 → effect 循环。

**修复**：改用 `useRef` 存储 `prevStars`，避免触发重渲染。

```tsx
const prevStarsRef = useRef(stars);
useEffect(() => {
  if (stars !== prevStarsRef.current) {
    prevStarsRef.current = stars;
    setShowSparkles(true);
    // ...
  }
}, [stars]);
```

### Home.tsx AnimatedNumber 性能优化 ✅

移除了 `useSpring` + `useEffect` 复杂依赖模式，简化直接显示格式化数字（父级 motion.div 已处理动画）。

## Supabase 建表 SQL 脚本修复（2026-04-28）

### 背景
为推进三端部署，需要创建 Supabase 数据库建表脚本（`supabase-schema.sql`）。
初始版本有多处 PostgreSQL 语法错误，已逐一修复。

### 修复记录

| 问题 | 根因 | 修复方案 |
|------|------|----------|
| `CREATE POLICY IF NOT EXISTS` 语法错误 | PostgreSQL 不支持该语法 | 改为 `DROP POLICY IF EXISTS` + `CREATE POLICY` |
| `auth.uid()::text::uuid` 冗余强转 | `auth.uid()` 本身返回 `uuid`，无需再转 | 全部改为 `auth.uid()` |
| `star_transactions.member_id` 类型为 `text` | 与 `members.id`（uuid）类型不匹配，外键无法建立 | 改为 `uuid + references members(id)` |
| `related_task_id` 等字段类型为 `text` | 同上，与对应表主键类型不一致 | 改为 `uuid + references 对应表(id)` |
