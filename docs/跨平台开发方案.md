# 愿望卡 - 跨平台开发方案

## 一、技术选型

| 平台 | 技术栈 | 开发方式 | 预估周期 |
|------|--------|----------|----------|
| **微信小程序** | Taro 3 + React + TypeScript | 从 Web 版迁移 | 2-3 周 |
| **Android** | Capacitor 5 (或 Tauri) | Web 壳 + 原生插件 | 1-2 周 |
| **iOS** | Capacitor 5 (或 Tauri) | Web 壳 + 原生插件 | 1-2 周 |

### 为什么选 Capacitor？

1. **代码复用率 95%+**：现有 React 代码几乎可直接使用
2. **原生能力**：通过插件访问相机、推送、生物识别、日历等
3. **统一部署**：一套代码同时生成 Android + iOS
4. **热更新**：支持 Web 资源热更新，无需重新审核

### 为什么选 Taro 做小程序？

1. **React 语法**：与 Web 版保持一致的开发体验
2. **组件迁移**：大部分 UI 组件可直接复用逻辑
3. **Supabase 兼容**：`@supabase/supabase-js` 支持小程序环境
4. **微信生态**：原生支持微信登录、支付、分享

## 二、微信小程序版

### 2.1 项目初始化

```bash
# 安装 Taro CLI
npm install -g @tarojs/cli

# 创建项目
taro init wishcard-miniapp --template react-ts

# 安装依赖
cd wishcard-miniapp
npm install @supabase/supabase-js i18next react-i18next date-fns
```

### 2.3 关键适配点

| Web API | 小程序替代 | 说明 |
|---------|-----------|------|
| `window.speechSynthesis` | 微信同声传译插件 `wx.createRecognizerContext` | 语音识别+TTS |
| `localStorage` | `Taro.setStorageSync` | 本地存储 |
| `BrowserRouter` | `Taro Router` | 路由 |
| `framer-motion` | 小程序动画 `wx.createAnimation` 或移除 | 动画 |
| `lucide-react` | 小程序 icon 组件或图片图标 | 图标 |
| `Notification API` | 模板消息 + 订阅消息 | 推送 |
| `navigator.mediaDevices` | `wx.chooseImage` / `wx.chooseMedia` | 相机 |

### 2.4 微信特有功能

- **微信登录**：`wx.login()` + Supabase 自定义 Auth Provider
- **订阅消息**：任务审批/完成时推送模板消息
- **分享**：支持分享到聊天、朋友圈
- **小程序码**：家庭邀请码 → 小程序码扫码加入

### 2.5 性能优化

- 分包加载：首页 < 2MB，其他页面按需加载
- 虚拟列表：任务列表使用 `<scroll-view>` + 虚拟渲染
- 图片懒加载：`<image lazy-load>`
- 骨架屏：首页加载时展示骨架屏

### 3.3 日历集成（原生）

```typescript
// src/lib/nativeCalendar.ts
import { Calendar } from '@capacitor/calendar';

export async function syncToNativeCalendar(tasks: Task[]) {
  // 请求权限
  const { granted } = await Calendar.requestPermissions();
  if (!granted) return;

  for (const task of tasks) {
    await Calendar.createEvent({
      title: task.title,
      location: '',
      startDate: new Date(task.startTime).getTime(),
      endDate: new Date(new Date(task.startTime).getTime() + 3600000).getTime(),
      isAllDay: false,
      alerts: [{ minutes: 15 }],
    });
  }
}
```

### 3.4 推送通知

```typescript
// src/lib/nativePush.ts
import { PushNotifications } from '@capacitor/push-notifications';
import { LocalNotifications } from '@capacitor/local-notifications';

export function initPushNotifications() {
  PushNotifications.requestPermissions();
  PushNotifications.register();

  PushNotifications.addListener('registration', (token) => {
    // 将 token 发送到 Supabase 存储以实现定向推送
    console.log('Push registration token:', token.value);
  });

  PushNotifications.addListener('pushNotificationReceived', (notification) => {
    console.log('Push received:', notification);
  });
}

// 本地定时提醒
export async function scheduleTaskReminder(taskId: string, title: string, at: Date) {
  await LocalNotifications.schedule({
    notifications: [{
      title: '任务提醒',
      body: `该完成任务啦：${title}`,
      id: parseInt(taskId.slice(-6), 36),
      schedule: { at },
      sound: undefined,
      attachments: undefined,
      actionTypeId: '',
      extra: { taskId },
    }],
  });
}
```

### 3.5 签名与发布

**Android:**
- 生成签名密钥：`keytool -genkey -v -keystore wishcard.keystore`
- 在 `android/app/build.gradle` 配置签名
- 构建：`cd android && ./gradlew assembleRelease`
- 上传 Google Play Console

**iOS:**
- Apple Developer 账号配置证书 + Provisioning Profile
- Xcode 打开 `ios/App` 目录
- 配置 Bundle ID、Capabilities（Push、Calendars）
- Archive → Upload to App Store Connect

## 四、代码共享策略

```
wishcard/                    # Monorepo 根目录
├── packages/
│   ├── shared/             # 共享代码包
│   │   ├── lib/
│   │   │   ├── api.ts      # Supabase API（Web/小程序/原生共用）
│   │   │   ├── types.ts    # 类型定义
│   │   │   ├── analytics.ts # 行为分析
│   │   │   ├── templateApi.ts
│   │   │   └── i18n/       # 国际化文本
│   │   └── package.json
│   ├── web/                # Web 版（当前项目）
│   ├── miniapp/            # 微信小程序版（Taro）
│   └── native/             # Capacitor 原生壳
├── supabase/
│   └── functions/          # Edge Functions
└── docs/
```

### 共享层

所有平台共用的代码放入 `packages/shared/`：
- API 封装（`api.ts`, `templateApi.ts`, `analytics.ts`）
- 类型定义（`types.ts`）
- 国际化文本（`i18n/`）
- 业务逻辑（星星计算、审批流程、日历生成）

### 平台适配层

每个平台有独立的适配代码：
- **Web**: `framer-motion` 动画, `lucide-react` 图标, Web Speech API
- **小程序**: `@tarojs/components`, 微信原生 API, 微信同声传译插件
- **原生**: Capacitor 插件, 原生推送, 原生日历

## 六、应用商店合规

### 微信小程序
- 用户隐私协议 + 数据安全说明
- 不含虚拟支付（心愿商城的"星星"不算真实货币，但需说明）
- 内容安全检测（用户输入的标题/描述需过审）

### Google Play
- 隐私政策 URL（必须）
- 数据安全声明（Supabase 服务器位置、数据类型）
- COPPA 合规（面向儿童的应用需特别注意）
- 目标 API Level 34+ (Android 14)

### App Store
- 隐私政策 URL + App Tracking Transparency
- 儿童类别需满足 App Store Review Guidelines for Kids
- 家庭共享支持
- 中国区需 ICP 备案号
