## 一、项目全景与工具分工

### 1.1 为什么需要三个工具？

```
┌─────────────────────────────────────────────────────────────────────────┐
│                           比乐时开发三工具链                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   Stitch (设计)          AI Studio (翻译)          Hermes (开发)        │
│   ┌─────────┐            ┌─────────┐              ┌─────────┐         │
│   │ 🎨 界面  │    →      │ 📐 组件  │     →        │ 💻 代码  │         │
│   │   设计   │            │   生成  │               │   实现  │         │
│   └─────────┘            └─────────┘              └─────────┘         │
│       ↓                      ↓                        ↓                │
│   交付：设计稿              交付：组件代码            交付：可运行App    │
│   (HTML/CSS)              (Flutter/Dart)            (完整项目)          │
│                                                                         │
│   Stitch 负责「看起来怎么样」                                            │
│   AI Studio 负责「怎么实现这些组件」                                     │
│   Hermes 负责「把一切组装起来运行起来」                                   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 1.2 三工具核心能力对比

| 维度 | Stitch | AI Studio | Hermes |
|:-----|:-------|:----------|:-------|
| **核心能力** | AI 生成设计稿 | 设计转代码 | 全栈代码开发 |
| **输入** | 文字描述 + 参考图 | Stitch 导出的设计 | 需求文档 |
| **输出** | HTML/CSS 设计稿 | Flutter/React 组件 | 完整可运行项目 |
| **擅长** | 快速原型、UI 探索 | 组件工程化 | 业务逻辑、架构设计 |
| **弱点** | 不能直接生成 App | 需要设计稿输入 | 设计能力弱 |
| **使用门槛** | 低 | 中 | 中高 |
| **使用频率** | 每个页面 1-2 次 | 每个页面 1 次 | 持续使用 |

### 1.3 你的准备工作清单

在开始之前，确保以下工具已安装配置好：

```
✅ Stitch
   - 访问：https://www.stitch.design 或对应的 IDE 插件
   - 验证：能正常打开并生成简单设计稿

✅ AI Studio（Claude Code / Cursor / Windsurf 任选其一）
   - 访问：https://claude.ai/code 或对应 IDE
   - 验证：能正常执行命令、读写文件

✅ Hermes（本地 AI 开发环境）
   - 安装 Ollama：brew install ollama
   - 安装 Whisper：见本文档「附录 A」
   - 验证：ollama list 能看到模型列表

✅ 开发环境
   - Flutter SDK 3.x：flutter --version
   - Xcode（Mac）：xcode-select -p
   - Android Studio 或 iOS Simulator
```

## 十、常见问题与解决

### Q1: Stitch 生成的样式不符合预期？

**解决**：使用「调整 Prompt」逐步修正
```
"调整[元素]：
- 当前问题：[描述]
- 期望：[描述]
```

### Q2: AI Studio 生成的组件有报错？

**解决**：提供更具体的代码规范
```
"使用 GetX：
- Controller 继承 GetxController
- 使用 Obx() 包裹响应式变量
- 不要使用 StatefulWidget"
```

### Q3: Ollama 模型推理很慢？

**解决**：
1. 使用更小的模型：`phi3:3.8b` 替代 `qwen2.5:7b`
2. 启用 GPU 加速
3. 减少 Prompt 长度

### Q4: Whisper 识别不准？

**解决**：
1. 使用 larger 模型（更准确但更慢）
2. 添加语言提示：`whisper --language zh`
3. 预处理音频：降噪、提高音量

### Q5: Hermes 忘记之前的决策？

**解决**：
1. 使用 `/memory` 查看记忆
2. 定期更新 MEMORY.md
3. 使用 `/title` 命名会话
